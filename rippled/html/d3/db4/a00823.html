If you hit this, then you've leaked one or more objects of the specified class; the name should have been printed by the line below.

If you're leaking, it's probably because you're using old-fashioned, non-RAII techniques for your object management. Tut, tut. Always, always use ScopedPointers, OwnedArrays, SharedObjects, etc, and avoid the 'delete' operator at all costs!
