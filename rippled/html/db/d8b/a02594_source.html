219 // export ROCKSDB_TESTS=PerfContextTest.SeekKeyComparison
220 // For one memtable:
221 // ./perf_context_test --write_buffer_size=500000 --total_keys=10000
222 // For two memtables:
223 // ./perf_context_test --write_buffer_size=250000 --total_keys=10000
224 // Specify --random_key=1 to shuffle the key before insertion
225 // Results show that, for sequential insertion, worst-case Seek Key comparison
226 // is close to the total number of keys (linear), when there is only one
227 // memtable. When there are two memtables, even the avg Seek Key comparison
228 // starts to become linear to the input size.
