1 ## Dependencies
2 
3 RocksDB is developed on Linux (CentOS release 5.2), with gcc 4.8.1.
4 It depends on gcc with C++11 support.
5 
6 * RocksDB depends on the following libraries:
7  - [zlib](http://www.zlib.net/) - a library for data compression.
8  - [bzip2](http://www.bzip.org/) - a library for data compression.
9  - [snappy](https://code.google.com/p/snappy/) - a library for fast
10  data compression.
11  - [gflags](https://code.google.com/p/gflags/) - a library that handles
12  command line flags processing.
13 
14 RocksDB will successfully compile without the compression libraries included,
15 but some things may fail. We do not support releases without the compression
16 libraries. You are on your own.
17 
18 ## Supported platforms
19 
20 * **Linux**
21  * Upgrade your gcc to version at least 4.7 to get C++11 support.
22  * Install gflags. If you're on Ubuntu, here's a nice tutorial:
23  (http://askubuntu.com/questions/312173/installing-gflags-12-04)
24  * Install snappy. This is usually as easy as:
25  `sudo apt-get install libsnappy-dev`.
26  * Install zlib. Try: `sudo apt-get install zlib1g-dev`.
27  * Install bzip2: `sudo apt-get install libbz2-dev`.
28 * **OS X**:
29  * Install latest C++ compiler that supports C++ 11:
30  * Update XCode: run `xcode-select --install` (or install it from XCode App's settting).
31  * Install via [homebrew](http://brew.sh/).
32  * If you're first time developer in MacOS, you still need to run: `xcode-select --install` in your command line.
33  * run `brew tap homebrew/dupes; brew install gcc47 --use-llvm` to install gcc 4.7 (or higher).
34  * Install zlib, bzip2 and snappy libraries for compression.
35  * Install gflags. We have included a script
36  `build_tools/mac-install-gflags.sh`, which should automatically install it.
37  If you installed gflags by other means (for example, `brew install gflags`),
38  please set `LIBRARY_PATH` and `CPATH` accordingly.
39  * Please note that some of the optimizations/features are disabled in OSX.
40  We did not run any production workloads on it.
41 
42 ## Compilation
43 `make clean; make` will compile librocksdb.a (RocskDB static library) and all
44 the unit tests. You can run all unit tests with `make check`.
45 
46 If you followed the above steps and your compile or unit tests fail,
47 please submit an issue: (https://github.com/facebook/rocksdb/issues)
